# Example: minimal-planning-project — Northwind Loyalty

The smallest valid PAIK project: a single `project.md`, `lifecycle: planning`, `components: []`
and `environments: []`. Nothing is invented — the one link to a not-yet-created Jira project is
left without a `url` and says so in `purpose`, instead of pointing at a plausible-looking fake
address.

This is the example to point at when someone asks "do I really need to fill in an entire
`paik/` folder before I can start using PAIK?" — no. Start at [`paik/project.md`](paik/project.md)
and add `components`/`environments` entries only once they exist.

See [`../simple-project/`](../simple-project/) for the same project once it's actually built.
