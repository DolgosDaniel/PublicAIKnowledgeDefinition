# Example: simple-project — TaskFlow Lite

The minimal instance of the PAIK v0.3 standard: one team, one repo, one API, two environments,
three files (`project.md`, `component.md`, `environments/*.md`) — everything else (ticketing,
wiki, secrets location) lives inline as `owner`/`links` fields rather than separate documents.
Start at [`paik/project.md`](paik/project.md).

See [`../complex-project/`](../complex-project/) for the multi-service version of the same
standard.
